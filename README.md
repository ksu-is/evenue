# evenue
An event venue-specific application that pulls from ticketing companies to provide venue-specific information, accessibility, point-of-sale, and marketing to users.

Working Development Plan

**Phase 1: Discovery & Planning**

**Define core features**:
*   Event notifications
*   Ticket purchasing
*   Venue walkthrough (360° view)
*   Social feed for user content
*   Private event booking
*   Merchandise and food/beverage promotions
*   Ticket resale integration
*   Downloadable wallpapers
  
**Research & feasibility**:
*   Investigate APIs (e.g., Ticketmaster, AXS, `ticketpy`)
*   Explore legal considerations for image rights (especially for wallpapers)
*   Identify target venues (e.g., Tabernacle in Atlanta)
*   **User personas & journey mapping**
*   **Select tech stack** (e.g., Flutter or React Native for cross-platform, Firebase or AWS backend)

**Phase 2: Design & Prototyping**

*   **UI/UX design**:
*   Venue homepage with event listings
*   Interactive seat selection with 360° view
*   Social feed layout
*   Booking and scheduling interface

**Phase 3: Core Development**

**Backend development**:
*   User authentication & profiles
*   Event data ingestion from ticketing APIs
*   Notification system (push notifications)
*   Media upload and moderation for social feed

**Frontend development**:
*   Venue walkthrough (possibly using WebGL or Unity integration)
*   Seat selection and ticket purchase flow
*   Social feed and grievance submission
*   Wallpaper download section

**Phase 4: Integration & Testing**

*  Integrate with ticketing platforms (e.g., Ticketmaster via `ticketpy`)
*  Implement 360° and Street View-like navigation
*  Test across devices (iOS, Android)
*  Security & privacy testing
*  Beta testing with a small user group

**Phase 5: Launch & Marketing**

*  Launch MVP** for a single venue (e.g., Tabernacle)
*   Collect user feedback
*   Iterate on features and UX
*   Develop marketing materials for venue partners and users
*   App Store & Google Play deployment
